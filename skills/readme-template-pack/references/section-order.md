# README Section Order

Use this order unless the user requests otherwise:

1. Project
2. TL;DR
3. Downloads
4. Snapshots
5. Learn More

Reasoning:

- `Project` explains the value quickly.
- `TL;DR` gives immediate action.
- `Downloads` supports users who want binaries first.
- `Snapshots` provides visual confirmation.
- `Learn More` keeps the README short while routing detail into docs.
