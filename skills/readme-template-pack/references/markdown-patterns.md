# Markdown Patterns

Use these patterns:

- A short title and one-line summary at the top.
- A fenced `bash` block for TL;DR commands.
- A single Markdown table for download channels.
- Standard Markdown image syntax for screenshots.
- Flat bullet lists for doc links.

Avoid:

- Nested lists
- Giant badge walls
- Long architecture sections
- Placeholder text when real repository data exists
